# StoryForge AI - 多智能体协作写作系统架构

## 整体工作流程

```
用户创建项目 → 选择内容类型 → 规划 → 写作 → 批评 → 质量评分 → [循环写作] → 导出
```

### 详细步骤

1. **用户初始化项目**
   - 输入项目名称、描述
   - 选择内容类型：`full_novel`（长篇小说）/ `short_story`（短篇小说）/ `script`（剧本）
   - 系统自动创建目录结构

2. **设定世界观与人物**
   - 用户填写`设定圣经`（世界观、人设、背景设定等）
   - 这部分信息会传递给后续所有 Agent 作为创作参考

3. **情节规划阶段**
   - 根据内容类型自动选择对应 Planner Agent
   - 输出：分卷/分章大纲，每章简要情节
   - 用户可以审核修改大纲

4. **逐章写作循环**
   - 每章由 Writer Agent 生成初稿
   - Critic Agent 评审，提出修改意见
   - （可选）Editor Agent 根据批评意见修改
   - Quality Agent 给章节打分
   - 自动保存到数据库，继续下一章

5. **导出**
   - 用户可以导出为 `txt` / `docx` / `html` / `epub` 格式

## Agent 职责分工

### 1. Planner - 情节规划师
**职责**：根据用户需求和设定，设计整体情节框架，分章列出大纲

**对应 Prompt 文件**：
- `planner.md` - **默认**，长篇小说
- `planner_short_story.md` - 短篇小说
- `planner_script.md` - 剧本/脚本

**输出**：JSON 格式的章节列表，包含章节标题和梗概

---

### 2. Writer - 正文作家
**职责**：根据规划大纲逐章生成正文内容，保持人设和情节一致性

**对应 Prompt 文件**：
- `writer.md` - **默认**，长篇小说
- `writer_script.md` - 剧本/脚本

**要求**：
- 遵守设定圣经中的人设世界观
- 符合前面章节已经写过的情节
- 保持文笔流畅，符合中文阅读习惯
- 完成当前章节的情节目标

---

### 3. Critic - 批评家
**职责**：对刚写完的章节进行评审，找出问题，提出具体修改建议

**对应 Prompt 文件**：`critic.md`

**评审维度**：
- 情节连贯性与合理性
- 人设一致性
- 文笔流畅度
- 节奏张弛
- 错别字/语法错误

**输出**：markdown 格式的评审意见 + 具体修改建议

---

### 4. Editor - 修改专家
**职责**：根据 Critic 的评审意见修改正文

**对应 Prompt 文件**：`editor.md`

**做法**：在原文基础上修改，保留好的部分，改进不好的部分

---

### 5. Quality - 质量评分员
**职责**：对章节进行多维度质量打分，输出综合评分和各维度分数

**对应 Prompt 文件**：`quality.md`

**评分维度**：
- 情节
- 人物
- 节奏
- 文笔
- 合规性

输出JSON格式评分，系统会保存评分用于统计和展示

---

### 6. Polish - 质量优化师
**职责**：根据检查出的问题，在不改变原剧情的前提下优化文字质量，去除AI模板化，修复情绪转换和悬念结尾

**对应 Prompt 文件**：`polish.md`

**做法**：在原文基础上优化，保留原剧情，只改进文笔和格式

---

### 7. Fix - 修复专家
**职责**：当任务执行失败（格式错误、解析错误）时，自动修复输出格式

**对应 Prompt 文件**：`fix.md`

---

### 7. Guardian - 架构守护者
**职责**：检查 Agent 输出是否符合指定的 JSON 格式要求，如果不符合要求指导 Agent 修正

**对应 Prompt 文件**：`guardian.md`

---

### 8. Compliance - 合规检查员
**职责**：检查内容是否违反合规政策（色情、暴力、政治敏感等）

**对应 Prompt 文件**：`compliance.md`

## 提示词文件对应关系总表

| 文件 | 对应 Agent | 适用体裁 | 作用 |
|------|-----------|----------|------|
| `planner.md` | Planner | 长篇小说 | 整体情节分章规划 |
| `planner_short_story.md` | Planner | 短篇小说 | 短篇小说情节规划 |
| `planner_script.md` | Planner | 剧本 | 剧本分幕规划 |
| `writer.md` | Writer | 长篇小说 | 生成正文 |
| `writer_script.md` | Writer | 剧本 | 生成剧本正文 |
| `critic.md` | Critic | 所有 | 评审章节，提出修改建议 |
| `editor.md` | Editor | 所有 | 根据批评修改正文 |
| `quality.md` | Quality | 所有 | 质量评分（输出JSON） |
| `polish.md` | Polish | 所有 | 质量优化修复（不改动剧情，只优化文字） |
| `fix.md` | Fix | 所有 | 修复格式错误 |
| `guardian.md` | Guardian | 所有 | 检查输出格式合规性 |
| `compliance.md` | Compliance | 所有 | 内容合规性检查 |

## 代码调用流程

```
1. 创建项目 → 启动生成任务
   ↓
2. Planner → 生成大纲
   ↓
3. 循环：对每个章节
   ↓
   ├─ Writer → 生成本章正文
   ↓
   ├─ Critic → 评审本章
   ↓
   ├─ Editor → 根据批评修改（可选）
   ↓
   └─ Quality → 打分保存
   ↓
4. 全部完成 → 可导出
```

## 设计理念

- **模块化**：每个 Agent 只做一件事，单一职责
- **体裁适配**：不同体裁有不同提示词，满足不同写作需求
- **可迭代改进**：写完一章评审一章，边写边改
- **用户可控**：用户可以随时停止、修改大纲、手动编辑
- **持久化**：所有章节和评分保存到数据库
