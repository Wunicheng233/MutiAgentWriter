连贯创作记忆

NovelState 动态状态追踪系统，自动记录角色、时间线、伏笔和文风变化。向量语义检索，智能关联前文内容，永不遗忘人物设定与剧情线索。

人机共创模式

支持策划方案和章节级确认机制。你掌控创作方向，AI 负责执行。不满意可以提出修改意见，系统按反馈定向优化，而非推倒重来。

结构化质量闭环

创作流程

从创意到作品

01

章节为最终叙事单位，不拆成互不连贯的小作文

02

Scene/span 只用于规划、诊断、定位和局部修复

03

局部修复必须携带前后邻接段，修后经 stitching 检查

04

连续两轮局部修复失败时，才升级为整章轻量重写

快速开始

五分钟启动你的创作环境

1

环境准备

需要 Python 3.10+、Node.js 16+、PostgreSQL 12+、Redis。

# macOS 安装 brew install postgresql@14 redis brew services start postgresql brew services start redis
2

安装依赖

# 创建虚拟环境 conda create -n storyforge python=3.10 conda activate storyforge # 后端依赖 pip install -r requirements.txt # 前端依赖 cd frontend && npm install && cd ..
3

配置环境变量

编辑项目根目录下的 .env 文件：

WRITER_API_KEY=your-volcano-engine-api-key-here DATABASE_URL=postgresql://postgres:postgres@localhost:5432/storyforge CELERY_BROKER_URL=redis://localhost:6379/0 CELERY_RESULT_BACKEND=redis://localhost:6379/0
4

初始化数据库

createdb storyforge alembic upgrade head
5

启动所有服务

需要三个终端窗口，都激活虚拟环境：

# 终端 1 - FastAPI 后端 uvicorn backend.main:app --reload --host 0.0.0.0 --port 8000 # 终端 2 - Celery Worker celery -A celery_app worker --loglevel=info # 终端 3 - Vite 前端 cd frontend && npm run dev

访问 http://localhost:5173， 注册账号，开始创作。

智能体团队

每个角色各司其职

Planner

生成小说整体策划、设定圣经、分章大纲、scene anchors

Context Assembler

汇总章节目标、scene anchors、前文、设定、风格和 NovelState

Writer

连续生成完整章节，按 scene anchors 推进但不拆段独立生成

Critic v2

多维度章节评审、打分、输出定位到 scene/span 的结构化问题清单

Failure Router

根据问题类型选择局部修复、stitching 或整章轻量重写

Revise

根据 Critic 或用户反馈，执行局部片段精准修复

Stitching Pass

修复过渡、指代、时间跳跃、情绪断裂和语气不一致

NovelState

追踪角色、时间线、伏笔、文风和世界观动态事实
