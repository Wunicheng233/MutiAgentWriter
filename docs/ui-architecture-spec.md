# StoryForge UI 架构设计规范
> Version: 1.0 | Date: 2026-04-25 | Status: 待评审

## 一、设计原则

### 1.1 核心哲学：「隐形的编辑器」

> 好的设计是看不见的。当作家专注于文字时，他们不应该注意到界面的存在。

- **极简优先**：所有非必要元素默认隐藏，需要时才出现
- **内容为王**：写作画布永远是视觉焦点，UI 元素服务于内容
- **渐进展示**：功能按上下文触发，而非永久占用屏幕空间
- **零视觉噪音**：避免不必要的边框、阴影、渐变、装饰性元素

### 1.2 布局原则：Claude Code / Zed 风格三栏架构

```
┌──────────┬────────────────────────────────┬──────────────────┐
│          │                                │                  │
│  导航栏  │        写 作 画 布             │   AI 助手面板    │
│  (60px)  │       (弹性宽度)               │   (可折叠)       │
│          │                                │                  │
│          │                                │                  │
│          │                                │                  │
│          │                                │                  │
│          │                                │                  │
│          │                                │                  │
│          │                                │                  │
└──────────┴────────────────────────────────┴──────────────────┘
```

**布局规则**：
- 左侧导航栏：固定宽度 60px，Icon-only，hover 展开标签
- 中间画布：弹性宽度，最大阅读宽度 720px（正文最佳阅读宽度），左右自动留白
- 右侧 AI 面板：默认宽度 320px，可完全折叠（宽度 0），也可拖拽调整
- 无顶部标题栏、无底部状态栏，状态信息融入对应区域边缘

## 三、三栏布局详细规范

### 3.1 左侧导航栏（Nav Rail）

**尺寸**：固定宽度 60px，100% 视口高度

**视觉规范**：
- 背景：`var(--bg-secondary)`
- 无右侧边框（用背景色差自然分隔）
- Icon 尺寸：20px × 20px
- Icon 间距：每个图标容器 44px × 44px（确保可点击区域足够）
- Icon 颜色：默认 `var(--text-secondary)`，激活 `var(--accent-primary)`

**交互规范**：
- 可完全折叠：宽度 60px（默认）→ 12px（折叠），支持 `⌘ + B` 切换
- hover 图标：背景 `var(--bg-tertiary)`，图标颜色变 `var(--text-primary)`
- 激活状态：背景 `var(--accent-primary)` + 12% opacity，图标 `var(--accent-primary)`
- 点击后：面板从左侧滑入（宽度 240px），覆盖在画布之上（非挤压）
- 支持键盘快捷键：`⌘ + 数字` 快速切换面板

**折叠状态**：
- 宽度缩小到 12px，仅保留极细边栏
- 鼠标移入边栏自动展开到 60px
- 画布自动扩展，利用额外空间

**功能图标（从上到下）**：
1. 项目总览（Project Overview）
2. 编辑器（Editor）
3. 章节列表（Chapters）
4. 角色设定（Characters）
5. 质量分析（Analytics）
6. 设置（Settings）

*所有图标使用纯 SVG outline 风格，无 emoji*

---

### 3.2 中间写作画布（Canvas）

**核心原则**：这是用户 90% 时间关注的区域，必须做到「零干扰」

**排版规范**：
- 最大宽度：720px（正文最佳阅读宽度）
- 左右留白：自动计算，确保画布居中
- 上下内边距：`64px`（顶部）、`128px`（底部，确保最后一行能提到视口中上部）
- 正文字号：`17px`
- 正文行高：`1.8`（文学作品需要更宽松的行高）
- 段间距：`1.2em`（空一行的视觉效果）
- 首行缩进：可选（中文写作习惯）

**专注模式（Focus Mode）**：
- 快捷键 `⌘ + Shift + F` 进入
- 左右两侧面板完全折叠（宽度 0）
- 画布宽度自动扩展到 900px
- 所有非编辑相关 UI 元素透明度降低到 10%
- 打字时自动隐藏光标外 3 行以外的文字（"打字机模式" 可选）

**画布状态**：
- 编辑中：显示光标、行高亮
- AI 生成中：生成文字逐字淡入，左侧有细进度条
- 建议模式：AI 建议文字用下划线 + 不同颜色标注，hover 显示接受/拒绝按钮

---

### 3.3 右侧 AI 助手面板（Assistant Panel）

**尺寸**：默认宽度 320px，可折叠到 0px，可拖拽调整（240px - 480px）

**视觉规范**：
- 背景：`var(--bg-secondary)`
- 无左侧边框（背景色差自然分隔）
- 可拖拽 resize 手柄：宽度 4px，hover 时显示 `var(--accent-primary)` 指示线

**面板状态**：

**状态一：折叠（默认）**
- 宽度：0px，完全不可见
- 画布自动扩展到最大宽度
- 可以通过 `⌘ + \\` 快捷键或点击右下角浮动按钮展开

**状态二：对话模式**
- AI 对话历史（时间倒序）
- 当前对话上下文预览
- 底部输入框（可发送指令、提问）
- 支持 Markdown 渲染、代码高亮

**状态三：建议模式**
- 显示当前章节的 AI 建议
- 按类型分组：情节建议、人物一致性、文笔优化、伏笔回收
- 每条建议可：接受、修改、忽略、添加到待办

**状态四：参考模式**
- 显示当前写作的上下文参考
- 前文摘要、人物状态、时间线、已埋伏笔
- 自动关联当前内容，实时更新

## 四、组件设计规范

### 4.1 按钮系统

| 类型 | 视觉规范 | 使用场景 |
|------|---------|---------|
| **主按钮** | 背景 `var(--accent-primary)`，文字白色，圆角 24px，无阴影 | 核心操作：开始生成、确认修改 |
| **次按钮** | 背景透明，文字 `var(--text-body)`，边框 `var(--border-default)`，圆角 20px | 次要操作：取消、跳过、查看详情 |
| **幽灵按钮** | 背景透明，无文字，仅 Icon，hover 显示背景 | 工具栏操作：撤销、重做、格式 |
| **浮动按钮** | 圆形 48px，背景 `var(--accent-primary)`，阴影 elevated | 右下角快速操作：展开 AI、新建章节

**主题切换**：内置在设置页面中，不作为导航栏常驻按钮 |

**按钮动画**：
- hover：`scale(1.02)`，背景色加深 5%
- active：`scale(0.98)`
- transition：`150ms ease-out`

---

### 4.2 卡片系统

**原则**：无圆角、无边框、无阴影，用背景色区分层级

| 层级 | 视觉规范 | 使用场景 |
|------|---------|---------|
| **一级卡片** | 背景 `var(--bg-secondary)`，圆角 12px | 重要内容容器：章节列表、设置面板 |
| **二级卡片** | 背景 `var(--bg-tertiary)`，圆角 8px | 次级内容：建议条目、历史记录 |
| **内联卡片** | 背景透明，hover 显示 `var(--bg-tertiary)` | 列表项、可点击条目 |

---

### 4.3 输入框系统

**原则**：极简，无默认边框，focus 时才显示边框

**规范**：
- 默认状态：背景 `var(--bg-tertiary)`，无边框，圆角 8px
- Focus 状态：边框 `var(--accent-primary)` 2px，背景 `var(--bg-primary)`
- Placeholder：`var(--text-muted)`，字号小 1px
- 无 label，用 placeholder 说明用途

## 五、动画与交互原则

### 5.1 动画时长规范

| 动画类型 | 时长 | 曲线 |
|---------|------|------|
| **面板展开/折叠** | 200ms | `ease-out` |
| **按钮状态变化** | 150ms | `ease-out` |
| **页面切换** | 250ms | `ease-in-out` |
| **内容渐入** | 300ms | `ease-out` |
| **AI 打字效果** | 每个字符 20-40ms 随机 | `linear` |

### 5.2 交互核心原则

1. **即时响应**：所有点击 100ms 内必须有视觉反馈
2. **无模态弹窗**：所有确认、提示都用非阻塞侧边滑入或 toast
3. **键盘优先**：所有常用操作都有快捷键，支持纯键盘操作
4. **无加载动画**：加载状态用进度条或骨架屏，不使用转圈 spinner
5. **微动画克制**：只在状态变化时使用动画，不添加装饰性动画

## 七、反模式清单（绝对禁止）

 **禁止**：添加装饰性图标、emoji、渐变背景
 **禁止**：使用圆角大于 12px 的容器
 **禁止**：添加永久显示的状态栏、标题栏
 **禁止**：模态对话框阻塞用户操作
 **禁止**：超过 300ms 的动画
 **禁止**：同时显示超过 2 个面板
 **禁止**：在画布区域添加任何非编辑内容

## 七、AI 对话机制讨论

### 7.1 是否需要增加通用 AI 对话？

**当前系统现状**：
- 现有 AI 功能是**工作流驱动**的：按固定流程执行 Planner → Writer → Critic → Revise
- 用户输入只限于：初始需求、章节级别的修改意见、确认/不确认
- 缺少**自由对话**能力，用户无法随时打断、提问、调整

**建议：增加通用 AI 对话机制 **

**核心价值**：
1. **创作过程的灵活性**：写一半突然想调整人设，不需要重启工作流
2. **探索式创作**："如果主角在这里选择 B 路线会怎样？"
3. **知识辅助**："这个历史时期的建筑是什么样的？"
4. **即时修改**："把这段对话改得更含蓄一些"

**实现方案（与现有工作流兼容）**：

```
┌─────────────────────────────────────────────────────────┐
│  右侧 AI 面板                                            │
├─────────────────────────────────────────────────────────┤
│  ┌───────────────────────────────────────────────────┐  │
│  │  Tab 1: 工作流状态  |  Tab 2: 自由对话            │  │
│  └───────────────────────────────────────────────────┘  │
│                                                         │
│  对话历史区域                                           │
│  - AI: "第3章已完成，主要情节发展是..."                │
│  - 用户: "把主角的反应改得更克制一些"                   │
│  - AI: "好的，已修改，更新后的段落是..."                │
│                                                         │
│  ┌───────────────────────────────────────────────────┐  │
│  │  输入框 + 发送按钮                                  │  │
│  └───────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
```

**对话模式**：
1. **上下文感知**：自动携带当前章节内容、已有人设、故事大纲
2. **指令分类**：
   - 润色类：改文笔、改对话、改节奏
   - 内容类：扩写、续写、调整情节
   - 咨询类：设定咨询、历史背景、写作建议
   - 控制类："暂停工作流"、"回到上一步"

**与现有工作流的集成**：
- AI 对话不打断主工作流，而是作为**并行辅助**
- 用户接受对话中的修改建议后，更新到编辑器
- 重大修改可以选择"基于此对话重启工作流"

**优先级建议**：Phase 3 完成右侧面板基础框架后，在 Phase 4 增加对话功能

## 八、验收标准

设计完成后，需要满足以下标准才能通过验收：

1. **空白屏测试**：打开编辑器，不做任何操作，屏幕上应该只有文字和光标，其他元素不超过 3 个
2. **专注模式测试**：进入专注模式后，非编辑元素可见度 < 10%
3. **键盘操作测试**：所有核心功能（切换面板、主题切换、专注模式）都能用键盘完成
4. **性能测试**：面板展开/折叠动画帧率 > 55fps
5. **视觉一致性**：所有元素间距、颜色、字号严格符合规范，无临时样式
