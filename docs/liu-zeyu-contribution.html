整个系统的"大脑"——让 StoryForge AI 不再每次从零开始，而是能从每次生成中总结经验、沉淀技能，越用越懂作者。

# ExperienceExtractor — LLM 驱动的根因分析 class ExperienceExtractor: def extract(self, signal: FeedbackSignal) -> WritingExperience: # 分析 Critic 低分的根因：是设定冲突？语气漂移？情节断裂？ root_cause = self._analyze_root_cause(signal) # 生成结构化改进建议 suggestion = self._generate_suggestion(root_cause, signal) return WritingExperience( problem_type=root_cause.type, evidence=signal.evidence, suggestion=suggestion, confidence=root_cause.confidence )
backend/core/skill_runtime/skill_assembler.py
Core · Python
SkillAssembler — Writer 每次生成前，根据当前章节的角色、剧情阶段、场景类型，从 ChromaDB 中语义检索最相关的 3-5 条技能，动态注入 Writer prompt。技能的来源有两个：预置的名家文风（鲁迅/刘慈欣等）和 Hermes 自动提炼的经验技能。
语义检索 ChromaDB 动态注入 Character Style Writing Style
# SkillAssembler — 上下文感知的技能注入 class SkillAssembler: def assemble(self, ctx: ChapterContext) -> list[Skill]: # 根据角色、剧情阶段、场景类型检索技能 query = self._build_query(ctx.characters, ctx.plot_stage, ctx.scene_type) skills = self.vector_db.similarity_search(query, k=5) # 按置信度排序，过滤低质量技能 return [s for s in skills if s.confidence >= 0.5]

frontend/src/hooks/useScrollReveal.ts
Frontend · TS
封装 IntersectionObserver 的通用 Hook，支持自定义 threshold 和 rootMargin。初始化时自动扫描已可见元素，防止首屏内容空白。提供 initScrollReveal 批量观察和 useScrollReveal 单元素 hook 两种使用方式。
IntersectionObserver scroll reveal custom threshold
// useScrollReveal — 滚动触发展入动画 export function useScrollReveal<T extends HTMLElement>(options?) { const ref = useRef<T>(null) useEffect(() => { const el = ref.current if (!el) return const observer = new IntersectionObserver( ([entry]) => { if (entry.isIntersecting) { el.classList.add('visible') observer.unobserve(el) } }, { threshold: 0.35, ...options } ) observer.observe(el) return () => observer.disconnect() }, []) return ref }
