**日期：** 2026-05-05

## 概述

本设计文档描述了 StoryForge AI 前端编辑器的重大体验升级，目标是从"AI 吐字的生成器"转变为"真正的 AI 写作协作者"。

核心设计原则：
- **AI 跟着用户走**：不是用户去问 AI，而是 AI 跟随用户的选区和上下文主动提供帮助
- **渐进式增强**：先用纯前端快速实现 MVP，收集反馈后再逐步优化后端支持
- **独立交付**：每个功能独立成 PR，按优先级逐个交付

## 模块 1：选区 AI 操作面板

### 功能描述

用户选中文本后，自动获得 AI 驱动的上下文相关操作。

### 交互设计

```
┌─────────────────────────────────────────────────────────────┐
│  编辑器正文                                                │
│                                                             │
│  这是一段用户选中的文本 → 弹出浮动工具栏                    │
│  ┌──────────────────────────────────────────────┐          │
│  │  润色   扩写   缩写   更多操作 →    │          │
│  └──────────────────────────────────────────────┘          │
│                                                             │
└─────────────────────────────────────────────────────────────┘

右侧边栏（选择"更多操作"后自动展开）：
┌─────────────────────────────────┐
│  选区智能操作                 │
│                                 │
│   角色语气改写                │
│     [ 李逍遥 ] [ 赵灵儿 ] [...] │
│                                 │
│   增加戏剧张力                │
│   插入伏笔                    │
│   检查剧情连续性              │
│                                 │
│  ┌───────────────────────────┐ │
│  │ 预览：AI 改写后的内容     │ │
│  │ （diff 高亮显示差异）     │ │
│  └───────────────────────────┘ │
│                                 │
│  [ 应用 ]  [ 放弃 ]             │
└─────────────────────────────────┘
```

### 技术实现

1. **选区检测**
   - 使用 TipTap 的 `selection` API 监听文本选择
   - 防抖处理（300ms），避免频繁弹出
   - 最小选区长度：5 个字符

2. **浮动工具栏**
   - 使用现有 `Popover` 组件（`components/v2/Popover`）
   - 按钮分为「常用操作」（显示在工具栏）和「高级操作」（展开到侧边栏）

3. **上下文注入**
   - 自动拼接 prompt：当前章节标题 + 前文 500 字摘要 + 选中的文本 + 具体操作指令
   - 复用现有的 `aiChat` API，无需后端修改（Phase 1）

4. **Diff 预览**
   - 使用 `diff` npm 包（`diff` 或 `diff-match-patch`）
   - 绿色高亮新增，红色高亮删除
   - 用户可逐句接受/拒绝

## 模块 2：版本历史与时光机

### 功能描述

从只能"恢复"升级为能"对比"、"分支"、"查看演变轨迹"的完整版本控制系统。

### 交互设计

```
版本历史侧边栏（右侧）：
┌─────────────────────────────────────────┐
│  版本历史  [+ 创建变体]               │
│                                         │
│   V3 - 2026-05-05 15:30  (当前)      │
│       "主角修改，增加冲突"              │
│       用户手动保存 · 2180 字            │
│                                         │
│  ⏪ V2 - 2026-05-05 12:15              │
│  [对比] [恢复] [→ 创建变体]             │
│       "初稿完成"                        │
│       AI 生成 · 2150 字                 │
│                                         │
│  ⏪ V1 - 2026-05-04 20:45              │
│  [对比] [恢复] [→ 创建变体]             │
│       "AI 生成初稿"                     │
└─────────────────────────────────────────┘

双栏对比视图：
┌───────────────────┬───────────────────┐
│  V2 (旧版本)      │  V3 (新版本)      │
│                   │                   │
│  他用右手拔剑     │  他用左手拔剑    │
│  走向门口         │  走向门口          │
│                   │  心里默念咒语    │
│  开门出去         │  开门出去          │
└───────────────────┴───────────────────┘
  [接受全部变更]      [恢复此版本]
```

### 变体章节功能

- 从某个历史版本创建变体：`章节 2` → `章节 2·变体 A`
- 变体在章节列表中用特殊图标标记
- 支持将变体"提升为正式版本"，替换原章节
- 变体之间可以互相 diff 对比

### 技术实现

1. **Diff 引擎（Phase 1）**
   - 前端加载两个版本的完整内容
   - 使用 `diff` 包计算字符级差异
   - 双栏同步滚动展示

2. **数据结构（后端需要小修改）**
   - 在 `chapter_version` 表增加：
     - `label`（可选版本名称/标签）
     - `is_variant`（是否是变体）
     - `parent_version_id`（变体父版本）

3. **时间线视图（Phase 2）**
   - 用 Canvas/SVG 绘制版本演变路径
   - 显示变体分支关系

## 模块 3：角色与设定 Bible

### 功能描述

独立的角色、世界观设定面板，与正文深度联动，支持一致性检测。

### 交互设计

```
左侧边栏（与大纲同区域，可拖拽切换标签）：
┌─────────────────────────────────────────┐
│  大纲   角色   世界观   待办   │
│                                         │
│   搜索设定...                         │
│                                         │
│  ▶ 李逍遥 (主角)            [编辑]      │
│     · 身份：蜀山弟子                    │
│     · 性格：乐观、仗义                  │
│     · 口头禅："吃到老，玩到老"          │
│                                         │
│     [ 用此角色视角重写选区 ]            │
│     [ 优化选区台词风格 ]                │
│                                         │
│  ▶ 赵灵儿 (女主角)                      │
│  ▶ 林月如 (女二号)                      │
│                                         │
│  ───────────────────────────────────   │
│   一致性提醒 (3)                      │
│  ┌───────────────────────────────────┐│
│  │ 第3章说左手剑，第5章右手剑         ││
│  │   查看冲突位置                   ││
│  └───────────────────────────────────┘│
│                                         │
│   世界观标签内容：                    │
│  ▶ 力量体系                             │
│     练气 → 筑基 → 金丹 → 元婴          │
│  ▶ 地理设定                             │
│     蜀山：位于川西，御剑飞行圣地        │
└─────────────────────────────────────────┘
```

### 智能联动功能

1. **实时角色高亮**
   - 正文中的角色名自动下划线高亮
   - 鼠标悬停显示设定摘要 tooltip
   - 点击跳转到角色面板

2. **一致性检测**
   - 每保存一次触发一次扫描（节流 10s）
   - AI 对比本章内容与设定库的冲突
   - 在侧边栏显示温和提醒，不打断写作

3. **快速操作**
   - 选中文本 → 一键"用 X 的视角重写"
   - 选中文本 → 一键"优化为 X 的说话风格"

### 技术实现

1. **设定数据解析**
   - 从策划方案 Markdown 中自动提取角色/世界观信息
   - 支持用户手动编辑、新增设定项
   - 设定存储在项目元数据中

2. **编辑器高亮**
   - 使用 TipTap 的 `Decorations` API
   - 正则匹配角色名，添加内联样式

3. **一致性检测（Phase 2）**
   - Phase 1 只做设定面板和快速操作
   - Phase 2 加入 AI 驱动的冲突检测

## 模块 4：键盘快捷键与无干扰模式

### 功能描述

增强 Focus Mode，提供更沉浸的写作体验。

### 功能列表

1. **Typewriter 模式**
   - 光标永远保持在屏幕中间
   - 滚动页面而不是移动光标
   - 视觉上更舒适，减少颈部运动

2. **Fade 模式**
   - 只有当前行/段落是 100% 不透明
   - 其他文字透明度降低到 30-50%
   - 帮助用户专注于正在写的内容

3. **命令面板 (Cmd+K)**
   - 类似 VS Code 的命令面板
   - 搜索所有编辑器功能
   - 不用鼠标就能完成所有操作

4. **Vim 键绑定（可选）**
   - 重度写作者的刚需
   - 可在设置中开关
   - 使用 `@tiptap/extension-vim`

### 技术实现

- 所有功能纯前端实现，无需后端
- 状态保存在 `useLayoutStore` 中
- 每个模式可以独立开关

## 模块 5：评论与协作

### 功能描述

虽然是单人写作工具，但用户需要给自己写备注、标记待办、以及接受 AI 的建议式修改。

### 功能列表

1. **选段评论**
   - 选中一段文字 → 添加评论/备注
   - 评论显示在右侧边栏
   - 正文中用黄色底纹标记有评论的段落

2. **待办标记**
   - 自动解析正文中的 `[TODO: xxx]` 标记
   - 在左侧边栏汇总所有待办项
   - 点击跳转到对应位置

3. **修订模式（AI 改写）**
   - AI 改写内容以"建议"形式呈现
   - 用户可以逐句接受/拒绝
   - 类似 Word 的修订模式，Git 的 diff 视图

## 模块 6：大纲与正文联动

### 功能描述

策划方案不是写完就扔，而是写作过程中的持续参考。

### 功能列表

1. **分屏视图**
   - 左侧大纲卡片墙，右侧正文编辑
   - 滚动同步：正文滚动到某章节，大纲对应高亮
   - 宽度可拖拽调整

2. **拖拽排序**
   - 拖动大纲节点调整章节顺序
   - 正文章节自动跟随调整
   - 操作确认弹窗，避免误操作

3. **偏离提醒**
   - AI 对比本章内容与策划大纲
   - 偏离较大时给出温和提醒
   - "这一章的主线冲突似乎和计划不太一样，需要调整吗？"

## 模块 7：离线与容错

### 功能描述

用户永远不会丢字，断网也能写。

### 功能列表

1. **LocalStorage 优先**
   - 每次按键都保存到 localStorage
   - 后台异步同步到服务器
   - 服务器冲突时提示用户选择版本

2. **离线模式**
   - 断网检测，显示小图标提示
   - 所有操作正常，只是不同步
   - 联网后自动增量同步

3. **崩溃恢复**
   - 精确记录光标位置、滚动位置
   - 页面崩溃重开后 100% 还原状态
   - 包括未保存的更改

4. **导出格式**
   - 支持：Markdown、纯文本、Word、PDF、EPUB、MOBI
   - 单章导出 / 全本导出
   - 可选是否包含评论和待办

## 模块 8：写作仪表盘与进度追踪

### 功能描述

长篇写作最大的问题不是"写不出来"，而是"坚持不下去"。用数据激励用户坚持。

### 功能列表

1. **每日字数统计**
   - 日历热力图（类似 GitHub 贡献图）
   - 7 天 / 30 天趋势图
   - 平均写作速度（字/小时）

2. **目标系统**
   - 设定每日目标（如"每天写 2000 字"）
   - 设定项目截止日期
   - 可视化进度条 + 预测完成时间

3. **连续写作 Streak**
   - 连续写作天数统计
   - "你已经连续写作 7 天了！"
   - 成就系统解锁

## 实现策略

### Phase 1：MVP 交付（第 1-3 模块）
- 纯前端实现，最小化后端改动
- 3-4 周完成核心体验
- 快速收集用户反馈

### Phase 2：完善优化（第 4-6 模块）
- 根据 Phase 1 反馈调整方向
- 补充必要的后端 API
- 2-3 周完成

### Phase 3：高级功能（第 7-8 模块）
- 离线同步可能需要较大架构调整
- 统计功能需要后端数据支持
- 2 周完成

## 技术栈说明

**现有技术栈保持不变：**
- React 19 + TypeScript
- TipTap 编辑器
- Zustand 状态管理
- TanStack Query (数据请求)
- Tailwind CSS

**新增依赖：**
- `diff` - 文本对比
- `@tiptap/extension-vim`（可选）- Vim 键绑定
- `chart.js` 或 `echarts` - 统计图表（模块 8）
