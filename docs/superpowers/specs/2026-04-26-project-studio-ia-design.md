# StoryForge AI - 项目工作室信息架构重构设计文档

> 日期: 2026-04-26
> 目标: 重构前端导航架构，实现"上下文感知的三栏布局"，打造以项目为中心的工作流体验

## 设计背景

当前产品的三栏布局骨架非常现代化，但导航层级存在以下体验问题：

1. **导航逻辑模糊**：NavRail 是全局的，但进入项目后项目内导航变成页面内 Tabs，层级不清晰
2. **"回书架"路径不明显**：用户在项目内时退出路径不够直观
3. **关键信息不可见**：项目生成状态需要在页面内容里找，不能一眼看到
4. **AI 面板没有上下文感知**：右面板虽然常驻，但不能感知当前页面内容

## 设计目标

 让用户永远清晰知道"我是否在某个项目里"
 减少导航层级，让项目内功能切换更顺滑
 关键状态（生成中、进度）永远可见
 为未来 AI 上下文感知打下基础
 改动量控制在 15% 以内，不推翻现有架构

## 信息架构

### 全局路由结构

```
/
├── /login - 登录页（公开）
├── /register - 注册页（公开）
├── /dashboard - 书架（全局）
├── /settings - 设置（全局）
└── /projects/:projectId - 项目工作室（以下都是项目内路由）
    ├── /overview - 项目概览
    ├── /outline - 大纲设定
    ├── /chapters - 章节列表
    ├── /editor/:chapterIndex - 编辑器
    ├── /analytics - 质量中心
    ├── /export - 导出分享
    └── /workflows/:runId - 工作流运行详情
```

### 导航上下文切换规则

**场景 A：不在项目内（书架/设置/登录）**
- NavRail 显示全局导航项：
  -  书架
  -  设置
- 顶栏：显示 "StoryForge AI" + 用户头像

**场景 B：在项目内**
- NavRail 显示项目内功能导航项：
  -  概览
  -  大纲
  -  章节
  -  编辑器
  -  质量中心
  -  导出分享
- 顶栏：[ 书架] + 《项目名称》 + [状态徽章] + [⋮ 更多菜单]

## 页面布局规范

### 统一布局结构

所有页面支持**三栏可折叠**设计，为用户提供最大的沉浸式创作空间：

```
  默认状态（全局/书架）                进入项目后默认状态
┌─────────────────────────────┐     ┌────────────────────────────────────────┐  ← 顶栏自动展开
│ ┌──────┐                    │     │ [ 书架]   《项目名称》   [ 57%]    │
│ │ Nav  │   主内容区          │     ├──────┬───────────────────────────┬─────┤
│ │ Rail │                    │     │ Nav  │                           │ AI  │
│ │展开  │                    │     │ Rail │     沉浸式创作区           │面板 │
│ │      │                    │     │展开  │                           │收起 │
│ └──────┘                    │     │      │                           │     │
└─────────────────────────────┘     └──────┴───────────────────────────┴─────┘
```

### 折叠行为规范

| 栏位 | 默认状态（全局） | 进入项目后 | 手动控制 |
|------|----------------|-----------|---------|
| **左侧导航栏** | 展开 | 保持展开 |  点击折叠按钮 |
| **顶部栏** | 收起 | **自动展开**（可设置关闭） |  双击顶部区域或快捷键切换 |
| **右侧 AI 面板** | 收起 | 保持收起 |  点击浮动按钮或快捷键呼出 |

### 视觉规范

1. **顶栏高度**：64px，底部 1px 边框
   - 收起时：高度变为 4px，鼠标悬停展开
   - 进入项目时：从 4px 平滑过渡到 64px

2. **侧导航宽度**：180px，图标 + 文字，选中项背景高亮
   - 收起时：宽度变为 56px，只显示图标
   - 折叠按钮位于导航栏底部

3. **AI 面板宽度**：320px
   - 收起时：显示右下角圆形浮动按钮
   - 点击按钮或快捷键（Ctrl+I）呼出

4. **主内容区内边距**：24px 32px
5. **Canvas 最大宽度**：1200px（内容居中）

### 设置项

在设置页面添加以下选项：

```
┌─────────────────────────────────────┐
│  布局偏好                            │
│                                     │
│  [x] 进入项目时自动展开顶栏          │

│  [ ] 默认展开 AI 助手面板            │
│                                     │
└─────────────────────────────────────┘
```

### 快捷键

- `Ctrl+B` / `Cmd+B`：切换左侧导航栏
- `Ctrl+T` / `Cmd+T`：切换顶栏
- `Ctrl+I` / `Cmd+I`：切换 AI 助手面板

### 顶栏组件细节

```
[ 书架] - 永远在左上角固定位置
《项目名称》 - 16px 字体，点击可以展开下拉快速切换最近项目
[状态徽章] - 显示生成状态：草稿/生成中/已完成/失败
[⋮ 更多菜单] - 包含：重置项目、删除项目、项目设置等
```

### ProjectOverview 拆分方案

**当前 1113 行的 ProjectOverview.tsx 拆分为 3 个独立页面：**

1. **概览页** `/overview`
   - 项目状态卡片
   - Agent 状态卡片
   - 当前工作流进度
   - 快速操作按钮

2. **大纲设定页** `/outline`（新）
   - 全书设定 bible
   - 角色列表
   - 分章大纲
   - 创作配置

3. **导出分享页** `/export`（新）
   - 导出格式选择
   - 分享链接管理
   - 协作者管理

## 状态徽章设计

显示在顶栏项目名称旁边，用户一眼就能看到项目状态：

| 状态 | 徽章样式 |
|------|---------|
| 草稿 |  灰色 "草稿" |
| 生成中 |  红色旋转动画 + 进度百分比 |
| 等待确认 |  黄色闪烁 "待确认" |
| 已完成 |  绿色 "已完成" |
| 失败 |  红色 "失败" |

## AI 面板上下文感知（基础版）

为未来演进打下基础，第一阶段实现以下感知能力：

1. **在概览页**：AI 助手显示"问我关于项目进度的问题"
2. **在大纲页**：AI 助手显示"帮我扩展设定、生成角色"
3. **在章节页**：AI 助手显示"选中文字可以让我改写、扩写"
4. **在编辑器**：AI 助手显示当前编辑章节的上下文提示

## 实施优先级

### Phase 1: 骨架重构（高优先级）
1. 改造 NavRail 支持上下文切换
2. 改造顶栏组件，添加项目状态和返回按钮
3. 调整路由结构，统一项目路由前缀

### Phase 2: 页面拆分（中优先级）
1. 拆分 ProjectOverview → 概览 + 大纲 + 导出
2. 更新所有页面内的导航链接

### Phase 3: 体验打磨（低优先级）
1. 项目切换下拉菜单
2. AI 面板上下文提示
3. 动画和过渡效果优化

## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 用户习惯了旧导航，新导航需要适应 | 中 | 添加新手引导 tooltip，高亮"回书架"按钮 |
| 路由变更导致书签失效 | 高 | 添加旧路由 → 新路由的重定向兼容 |
| ProjectOverview 拆分引入 bug | 高 | 保持旧组件可用，分步骤迁移功能 |
| 改动范围超出预期 | 中 | 严格按 Phase 实施，每阶段完成后验证 |

## 验收标准

- [ ] 用户从书架进入项目后，NavRail 自动切换为项目内导航
- [ ] 左上角"返回书架"按钮在所有项目内页面都可用
- [ ] 项目生成状态在顶栏实时更新
- [ ] 所有旧路由能正常重定向到新路由
- [ ] ProjectOverview 拆分为 3 个独立页面，文件大小 < 400 行/个
- [ ] 所有现有功能保持可用，无回归
