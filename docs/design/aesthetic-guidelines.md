# StoryForge AI 页面审美指南

> 本文档记录了用户在页面打磨过程中体现的审美偏好和设计原则，用于指导后续页面优化。

---

##  核心设计原则

### 1. **极简主义：移除所有冗余文字**
-  **删除解释性描述**：不要用大段文字告诉用户"这是什么"，组件应该自解释
-  **避免引导文案**：如"开始生成后，章节会在这里出现"这类提示可以压缩或移除
-  **标题即表达**：好的标题和视觉设计本身就能说明功能，不需要额外解释

**反面教材（已移除）：**
```
"从创意到作品交付的关键路径收成一张总览"
"开始生成后，这里会开始沉淀运行记录"
```

---

### 2. **视觉一致性：徽章、字体、间距统一**

#### Badge 变体使用规范
| 状态 | 变体 | 颜色 | 使用场景 |
|------|------|------|----------|
| 已完成 / 成功 | `success` |  绿色 | 生成完成、任务成功、步骤完成 |
| 运行中 / 活跃 | `status` |  蓝色 | 当前运行、进行中状态 |
| 等待确认 | `warning` |  黄色 | 等待人工确认 |
| 失败 / 错误 | `error` |  红色 | 任务失败、错误状态 |
| 次要信息 | `secondary` |  灰色 | 模式标签、类型标签 |
| **禁止** | `agent` |  紫色 |  不要用于"已完成"状态 |

#### 字体使用规范
- `font-serif`：仅用于**文章标题**、**章节内容**等文学性文本
- `font-medium`：用于所有**UI 标题**（卡片标题、智能体名称、按钮文字）
- **不要**用 font-serif 来强调 UI 元素名称

#### 间距规范
- 卡片内边距：使用统一的 `p-5` 或 `p-6`，不要出现过大的 `p-12`
- 元素对齐：标题、标签、内容区域要在同一垂直线上

---

### 3. **布局平衡：拒绝不对称**

#### 顶部容器设计原则
-  **左右权重平衡**：左侧信息区和右侧操作区视觉重量相当
-  **进度条横向展开**：不要让进度条偏安一隅
-  **操作按钮集中放置**：所有按钮放在顶部操作区，不要散落在页面底部

#### 网格布局原则
-  **等分网格**：`grid-cols-5`、`grid-cols-4` 等分布局优先
-  **对称响应**：移动端堆叠时也要保持居中对齐
-  **标签居中**：Badge、状态标签居中显示，不要左/右对齐

---

### 4. **交互聚焦：一个区域一个主要操作**

#### 卡片点击区域
-  **主点击**：卡片整体点击 = 主要操作（如：进入章节编辑）
-  **次级操作**：用 `...` 下拉菜单收纳，不要并列显示多个按钮
-  **事件隔离**：下拉菜单必须 `stopPropagation()`，防止触发卡片主点击

#### 按钮去重原则
-  **功能唯一**：同一页面不要出现"查看章节"和"章节"两个按钮
-  **条件渲染**：主按钮已覆盖某功能时，次要按钮自动隐藏
  ```tsx
  {runSummary.ctaLabel !== '查看章节' && (
    <Link to={`/projects/${projectId}/chapters`}>
      <Button variant="secondary" size="sm">章节</Button>
    </Link>
  )}
  ```

---

### 5. **数据真实：拒绝展示不合理状态**

#### 进度计算
-  **已完成项目强制 100%**：不管底层任务进度如何，`status === 'completed'` 时显示 100%
-  **边界值保护**：`Math.min(progress * 100, 100)` 防止超过 100%

#### 数据过滤
-  **过滤第 0 章**：前端展示和后端统计都要排除 `chapter_index = 0`
-  **合理数值**：避免出现"4/3"这种完成数大于总数的统计错误

#### 参数验证
-  **精确比较**：数字 `0` 是有效值，不要用 `if (!value)` 判断
  ```tsx
  //  错误：0 会被当作 falsy
  if (!chapterIndex)
  
  //  正确：精确比较
  if (chapterIndex !== undefined && chapterIndex >= 0)
  ```

---

### 6. **品牌感：标题与细节精致化**

#### Logo / 品牌标题
-  **全大写**：`uppercase` + 超大字间距 = 高级感来源
-  **字间距**：`tracking-[0.24em]`（关键！）
-  **次级色**：`text-[var(--text-secondary)]`，低调内敛
-  **小字号**：`text-sm`，精致不张扬
-  **禁止**：大字号 + 主色 + 衬线字体（太笨重）

**正确示例**：
```tsx
<span className="font-medium text-sm uppercase tracking-[0.24em] text-[var(--text-secondary)]">
  MultiAgentWriter
</span>
```

#### 侧栏圆角
-  **左侧栏**：右上角 + 右下角圆角 `rounded-tr-lg rounded-br-lg`
-  **右侧栏**：左上角 + 左下角圆角 `rounded-tl-lg rounded-bl-lg`
-  **效果**：栏与栏之间过渡柔和，不再生硬

#### 导航图标选中状态
-  **高对比度**：`bg-[var(--accent-primary)]` 实色背景 + `text-white` 白色图标
-  **层次感**：添加 `shadow-sm` 让选中状态浮起
-  **禁止**：同色系低对比度（绿色背景 + 绿色图标 = 看不见）

##  页面优化 Checklist

优化任何页面时，按此顺序检查：

###  第一步：内容精简
- [ ] 删除了所有解释性、引导性的冗余文字？
- [ ] 移除了"这是什么"、"你可以在这里"类描述？
- [ ] 每个区域只有一个标题，没有副标题/描述？

###  第二步：视觉统一
- [ ] Badge 变体使用正确（success=已完成，不要用 agent）？
- [ ] 字体使用正确（UI 标题用 font-medium，不用 font-serif）？
- [ ] 所有标签、Badge 居中对齐？
- [ ] 卡片间距统一，没有过大的内边距？

###  第三步：布局平衡
- [ ] 顶部容器左右对称，没有明显偏斜？
- [ ] 操作按钮都在顶部，没有散落在底部？
- [ ] 网格列数合理，元素分布均匀？

###  第四步：交互正确
- [ ] 下拉菜单点击不会触发卡片主跳转？
- [ ] 没有重复功能的按钮？
- [ ] 数字 0 不会被误判为无效参数？

###  第五步：数据真实
- [ ] 已完成项目进度显示 100%？
- [ ] 第 0 章已过滤？
- [ ] 统计数字合理（不会出现 4/3）？

##  反模式黑名单

这些设计出现时必须修复：

1. **解释性文字强迫症**："这个卡片用来显示 XXX"
2. **按钮重复症**：同一功能出现两个不同名称的按钮
3. **底部按钮孤儿**：操作按钮孤零零放在页面最底部
4. **font-serif 滥用**：UI 标题用衬线字体假装"有文化"
5. **紫色已完成**：用 agent 变体（紫色）显示"已完成"
6. **事件冒泡漏网**：点击下拉菜单却跳转到别的页面
7. **数字 0 恐惧症**：把有效值 0 当成 falsy 处理
8. **emoji/3D 图标滥用**：空状态、占位符使用 3D emoji、彩色卡通图标（齿轮、堆叠书本等）—— 必须改用统一风格的 SVG 线条图标或纯文字
